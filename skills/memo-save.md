# /memo-save — 保存新经验到知识库

## 触发词
`/memo-save`

## 核心流程

1. **定位 vault**：读取 `~/.memonaut.json` 获取 `vaultPath`，若不存在则使用 `~/memonaut-vault`
2. **引导填写**：询问用户以下信息：
   - 标题（简短描述）
   - 问题/背景
   - 解决方案
   - 注意事项
   - 标签（关键词，逗号分隔）
3. **AI 自动分类**：根据内容判断分类
   - `pitfalls/` — 踩坑记录（问题和教训）
   - `standards/` — 个人规范（最佳实践）
   - `solutions/` — 方案沉淀（完整解决方案）
4. **生成 Markdown**：使用以下模板生成文件内容
5. **用户确认**：展示生成的内容和目标路径，等待用户确认
6. **写入文件**：写入 `{vaultPath}/experiences/{分类}/{文件名}.md`
7. **更新索引**：运行 `node {项目路径}/scripts/memo-index.mjs` 更新索引

## 文件模板

```markdown
# [标题]

- **日期**: YYYY-MM-DD
- **标签**: 关键词1, 关键词2
- **场景**: 简述适用场景

## 问题/背景
## 解决方案
## 注意事项
```

文件名规则：标题转为简短的中文或英文短横线连接，如 `docker-启动超时.md`
